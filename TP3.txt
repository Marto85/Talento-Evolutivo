Objetivos específicos:
Funcionalidad: Asegúrate de cubrir todos los requisitos establecidos en el
enunciado y que la aplicación funcione de acuerdo a lo solicitado.
● Componentes técnicos
● Desarrollo construida con Node.js completo con intervenciones de Ing. de
Software (En caso de no haber cursado la asignatura correspondiente, se
deberá realizar un análisis explícito del alcance del sistema)
● Ruteo y controladores bien organizados.
● Manejo de datos con base de datos (MongoDB).
● Modelo de datos coherente (entidades, relaciones, validaciones).
● Autenticación y autorización (JWT, sesiones, Passport, etc.).
● Manejo de errores y middlewares personalizados.
● Uso de variables de entorno y configuración separada.
● Buena estructura de carpetas acorde al patrón utilizado (MVC)
● README completo con instrucciones de uso y dependencias.
Extensibilidad: Considera posibles características adicionales que puedan añadir
valor (pero no obligatorias), y que estén alineadas con los contenidos de la

críticos del sistema. Usa técnicas de testing vistas en clase (No es necesario

Mantenimiento y Evolución: Explicar que corrigieron errores para adaptar el
software a nuevas necesidades como mejorar rendimiento, seguridad y
escalabilidad.

Código

que siga buenas prácticas de desarrollo (como nombres claros para variables
y funciones, modularización, reducción de redundancias, entre otros).
● Comentarios: Incluye comentarios mínimos y precisos en las partes críticas del
código para poder defenderlo sin depender de ello.
● Repositorio y Entrega del Código: Sube el código a GitHub, tener un
README claro. Prepara también un archivo comprimido con el código para la
entrega en la plataforma.
● En cuanto a agregar otros módulos no vistos en clase, se podrá siempre y
cuando se explique en la documentación el motivo y porque se implementó.
